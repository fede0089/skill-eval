# Specification: Functional Testing Command

## Objective
Add a new subcommand `skill-eval functional --skill <path>` to evaluate if the results generated by a skill meet the `expectations` defined in the `evals/evals.json` file.

## Requirements
- The user executes the CLI from any project directory.
- The target skill is provided via the `--skill <path>` argument.
- The evaluations are defined in `<skillPath>/evals/evals.json`.
- The existing `trigger` command must remain intact and functional.
- The evaluation process for each eval:
  1. Execute Gemini with the `prompt` of the eval.
  2. Capture the result of the execution.
  3. Evaluate each of the `expectations` defined for the eval.
  4. Determine if each expectation is met (Pass/Fail).
  5. Accumulate the results in the standard format used by the tool.
- Artifacts from the run must be persisted in the `.project-skill-evals/runs/<timestamp>/` directory, similar to the `trigger` command.

## Technical Approach
- Add a new `functional` command to Commander.js in `src/commands/`.
- Extend or create a new evaluator core module `src/core/evaluator.ts` (or similar) that handles reading `evals.json`, executing the agent, and validating expectations.
- Ensure the result dumping mechanism is reused or adapted to support the functional testing outputs.
